学习工作丨MemoryCenter —— AI Agent 的时序记忆基础设施（5 天迭代成果）

## 正文

### 一句话介绍

**MemoryCenter** 是一个 Rust 单二进制的 AI Agent 时序记忆基础设施，让 LLM 拥有跨会话、跨 Agent、跨时间的持久记忆。开发者一键接入（配置 MCP），Agent 即自动获得归档、检索、淘汰能力——无需手动整理笔记，长项目关键决策不再被遗忘。

- **代码仓库**：https://github.com/LINGTIAN303/MemoryCenter

- **API Key**：`trae-contest-demo-key-2026`（评委测试用，请勿压测）

### 核心数据：评测指标

#### 1. LoCoMo 长时记忆基准（纯算法 F1 评分，无 judge 主观性）

| 模型 | baseline F1 | memory_center F1 | 提升 |
|------|-------------|----------|------|
| SenseNova | 0.1036 | **0.1465** | **+41.4%** |
| Step | 0.1105 | **0.1345** | **+21.7%** |

- **多跳推理（category_1）提升最显著**：+25% ~ +48.7%
- **大规模对话优势更明显**：conv-41（663 turns）Step 提升 **+94%**

#### 2. LongMemEval R@5 检索评测（500 题数据集 30 题抽样）

| 指标 | 结果 |
|------|------|
| recall_any@5 | **100%**（30/30） |
| recall_all@5 | 90%（27/30） |
| nDCG@5 | 0.9405 |
| 检索模式 | Hybrid（BM25 + 语义向量 RRF 融合） |

- **关键改进**：将原始对话 turns 内容加入索引后，R@5 从 **0% 跃升至 100%**——证明完整对话保真归档是检索精准命中的前提

#### 3. 上下文压缩优势（V2.36 Qwen2.5-7B）

| 条件 | 耗时 | 说明 |
|------|------|------|
| baseline | 127.1s | 全量对话历史发送给 LLM |
| memory_center | 87.6s | 归档 + 摘要 + retrieve 完整内容注入 |
| **速度提升** | **31%** | 上下文压缩带来显著加速 |

### 四个独家护城河

1. **三级索引周期 + 4 维加权评分淘汰**（天/周/月）—— 类比人脑记忆巩固机制
2. **完整对话非摘要归档**（竞品都走压缩/抽取路径）—— 检索可精准命中原始证据
3. **Rust 单二进制 + C ABI 嵌入宿主进程**（市场唯一可嵌入方案）—— 零外部依赖，9 种语言绑定
4. **17 类消息级标签**（粒度最细）—— LLM 可按需精准检索

### 创新点说明

#### 创新点 1：完整对话非摘要归档

不同于 Zep/Letta 走摘要抽取路径，MemoryCenter 保留原始对话 turns（含 thinking / tool_calls / attachments），让检索能精准命中证据 session。V2.35 评测显示：**turns 内容加入索引后 R@5 从 0% 跃升至 100%**。

#### 创新点 2：阈值监控 + 主动清空（v2.47）

跳过 LLM 压缩步骤，直接清空 LLM 内部会话上下文：
1. sidecar 监控 tokens 累积值（step-finish 真实消耗）
2. 阈值到达前主动归档完整上下文到 MemoryCenter
3. 插入 compaction 消息对，让 OpenCode 下次加载时无感清空旧消息
4. LLM 通过 prompt 工具召回历史记忆

这是"用 memory-center 归档摘要替代 LLM 压缩摘要"的核心机制。

#### 创新点 3：AgentAdapter trait 抽象层（v2.46）

新增 Agent（如 Claude Code / Cursor）只需：
1. 实现 `impl AgentAdapter for XxxDb`（4 个方法）
2. main.rs match 分支加一个选项

无需修改 watcher / archive / state 任何代码。
